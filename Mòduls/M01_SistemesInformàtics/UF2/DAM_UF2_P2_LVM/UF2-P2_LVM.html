Pràctica LVM

Part I - Creació de LVM

1. Partim d'una màquina virtual amb Linux (Debian o Ubuntu) instal·lat.

2. Afegir 2 discs: 5 GiB i 8 GiB.

3. Instal·lar LVM2

        sudo apt update
        sudo apt-get install lvm2  == sudo apt install lvm2

4. Particionar el segon disc afegit en dos (4+4 GiB)

1. Seleccionar dispositiu (/sdc) > crear tabla particiones: msdos

2. Añadir partición (Fer-ne 2 de 4096 MiB)

3. Aplicar canvis (check verd)

5. Crear volums físics amb el disc1 i la primera partició del disc 2.

sudo pvcreate /dev/sdb /dev/sdc1

6. Crear grup de volums amb el disc1 i, posteriorment, extendre el grup amb l'altre volum físic.

  1. vgcreate vg1 /dev/sd…

Creem el Grup de volums i l'anomenen vg1.

$ sudo vgcreate vg1 /dev/sdb /dev/sdc1

  1. vgextend vg1 /dev/sd…

Si ens deixem algun Grup de Volums, més endavant podem extendre’l amb:

$ sudo vgextend vg1 /dev/sdc2

Per veure l’estat dels Volums físics:

$ sudo pvscan 

7. Visualitzar els volums físics amb pvdisplay i el grup de volums amb vgdisplay.

8. Crear volums lògics anomenats audio i video amb lvcreate. El volum audio tindrà 1GiB i el de vídeo 2 GiB.

Creem dos Volums Lògics que anomenarem audio i video respectivament a partir del Grup de Volums vg1 anterior:

$ sudo lvcreate -L 1G -n audio vg1

    Logical volume "audio" created

$ sudo lvcreate -L 2G -n video vg1

    Logical volume "video" created

On l’opció:

  • L: Es el tamaño del Volumen Lógico.
  • n: Va seguido del nombre que le daremos a nuestro Volumen Lógico.

Observem l’estat dels Volums lògics creats:

*PE Size, cada cop que hi hagi un espai lliure al disc, l’agafarà de 4 en 4MiB.

9. Donar format als volums lògics creats amb el sistema ext3.

En crear els Volums Lògics audio i video, el sistema ha creat els dispositius /dev/vg/audio i /dev/vg/video que també es poden nombrar com /dev/mapper/vg-audio i /dev/mapper/vg-video respectivament.

Aquests seran els equivalents a les nostres particions de sempre, i que ara els podrem donar format per poder desar-hi arxius a dins:

$ sudo mkfs.ext4 /dev/vg/audio

$ sudo mkfs.ext4 /dev/vg/video

Per tant, ara ja podem crear els punts de muntatge i configurar el sistema per tenir-los muntats de manera permanent. Per això, afegim les línies següents al fitxer /etc/fstab:

$ sudo gedit /etc/fstab

/dev/vg/audio /audio ext4 rw,user 0 0

/dev/vg/video /video ext4 rw,user 0 0


10. Muntar els volums lògics audio i vídeo a les carpetes /audio i /video respectivament. (Crear primer aquestes carpetes)

Només ens queda crear els punts de muntatge /àudio i /vídeo i ja podrem accedir als nous Volums Lògics igual que fèiem abans amb les particions.

$ mount /dev/vg/audio

$ mount /dev/vg/video

Comprovar que realment tenim muntats aquests volums lògics i la seva mida amb df -h.

11. Ampliar el volum lògic video per donar-li 3GiB més del que té ara.

Ara, quan vulguem ampliar la mida d'un Volum Lògic ho farem amb l'ordre lvextend:

$ sudo lvextend -L +3G /dev/vg/video

$ sudo resize2fs /dev/vg/video

Comprovar que s'ha ampliat.

$ sudo lvscan

12. Escriure un fitxer a /audio i deixar aquest volum lògic a la meitat de la seva capacitat actual.  Comprova que s'ha fet més petit amb df -h i que el fitxer encara hi és.

Podem reduir la mida d’un Volum lògic amb lvreduce i retornar l’espai alliberat al Grup de volums:

$ sudo lvreduce -L -1G /dev/vg/video -r

Do you want to unmount "/video"? [Y|n] y

fsck de util-linux 2.20.1

/dev/mapper/vg-video: 11/262144 files (0.0% non-contiguous), 33870/1048576 blocks

resize2fs 1.42.9 (4-Feb-2014)

Resizing the filesystem on /dev/mapper/vg-video to 786432 (4k) blocks.

The filesystem on /dev/mapper/vg-video is now 786432 blocks long.

  Reducing logical volume video to 3,00 GiB

  Logical volume video successfully resized

En aquest cas, el sistema d’arxius s'ha de desmuntar abans de fer l'operació. Un cop finalitzada, el sistema torna a muntar-la de nou. El sistema realitza aquestes operacions per nosaltres automàticament.

13. Verifica el contingut total del grup de volums amb vgdisplay i el de cada volum lògic amb lvdisplay.

$ sudo vgdisplay
$ sudo lvdisplay

14. Verifica l'espai utilitzat de cada volum físic amb pvscan.

15. Crea un altre volum lògic anomenat dades amb la opció -i de lvcreate. Copiar un fitxer d'uns quants MiB i verificar quan hi ha ocupat de cada volum físic.

Part II - Instal·lació d'Ubuntu Server amb LVM

En aquesta part de la pràctica instal·larem el sistema sencer en una màquina amb LVM.

Utilitzarem el CD d'instal·lació d'Ubuntu Server.

16.- Crea una màquina virtual nova amb un disc de 20 GiB.

17.- Inicia l'instal·lador de Debian i, quan arribis a la següent pantalla de l'instal·lador:

Crea al disc de 20 GiB un sistema LVM amb els següents volums lògics:

  • 200MiB per a boot
  • 400MiB per a home
  • 6GiB per a root
  • 3GiB per a var
  • 512MiB per a swap

Dóna format a les particions que ho requereixin amb el tipus ext4. → (ext4 transaccional, es treballa a nivell transaccional per assegurar-nos que no es deixa una operació a mitges)

18.- Instal·la el sistema.

19.- Arrenca el sistema i comprova l'estat amb les comandes: df -h, pvscan, vgdisplay i lvdisplay.

20.- Amplia el volum lògic on està /home en un 1GiB més.

sudo lvextend -L +1G /dev/debian-vg/home

Comprova que ha funcionat l'ampliació.

21.- Apaga la màquina i afegeix un disc nou de 100 GiB al sistema.

config. > emmagatzemament > afegir disc dur

Torna a arrencar i crea una partició del nou disc que contingui la totalitat de la seva capacitat.

Cal donar-li format al nou disc per poder treballar-hi:

22.- Raona com ho faries per traslladar el contingut complet del disc de 20 GiB al nou de 100 GiB.

El nou Volum físic (/sdb) ha d’estar inclòs al mateix Grup de Volums que l’antic (/sda → debian-vg):

  1. Afegir el nou disc al Grup de Volums que hi ha creat: sudo vgextend vg1 /dev/sdc2

  1. Comprovar Volums físics:  sudo pvscan

Fes el traspàs d'un disc a l'altre. Quins problemes creus que et trobaràs per aconseguir que el nou disc arrenqui? doncs que no conté un MBR Com ho pots solucionar? executant grub-install /dev/sdb

El MBR es troba al disc /sda. Amb el move només es mou la partició /sda1, no la taula de partició MBR…

/sda

/sdb

MBR

—----->

sdb1

sda1


Nota 1: incorpora la partició del segon disc al grup de volums i fes servir la comanda pvmove per moure el contingut del disc inicial al nou disc.

  1. La migració de les dades es pot realitzar amb l’ordre pvmove: sudo pvmove /dev/sda1 /dev/sdb1

  1. Comprovar si s’ha mogut fent un sudo pvscan

Nota 2:executa grub-install /dev/sdb per fer que el disc nou arrenqui sol.

Per poder eliminar el /sda, haig de moure el MBR que conté al nou disc /sdb, perquè sinó el nou disc /sdb no arrencaria…  ←Si no elimino el primer disc (/sda) que conté el MBR, no caldria crear un nou MBR al segon disc (/sdb) ja que el sistema arrenca per ordre de discs:

  1. /sda
  2. /sdb
  3. …

Ara els 2 discs tenen un MBR, es crea un nou (no es un copy):

/sda

/sdb

MBR

—----->

MBR

sda1+

sdb1

el /sda té el seu MBR (pot arrencar), però està buit. O sigui que es pot esborrar.

23.- Un cop fet el traspàs de les dades, elimina el disc antic del sistema LVM.

El disc /dev/sdb queda alliberat de dades, per tant ja pot ésser eliminat del Grup de Volums:

$ sudo vgreduce debian-vg /dev/sda1

  Removed "/dev/sda" from volume group "debian-vg"


També podem eliminar  la seva etiqueta com a ''Volum físic'':

$ sudo pvremove /dev/sda1

  Labels on physical volume "/dev/sda" successfully wiped


Ara aquest disc ja pot ésser extret de la màquina.

Anota la sortida de les comandes: df -h, pvscan, vgdisplay i lvdisplay.

Apaga la màquina i retira el disc vell de forma definitiva.

Vaig a paràmetres de la MV i elimino el disc de 100GB anteriorment creat

Retirar el primer disc: botó dret a sobre Debian_LVM.vdi > Eliminar conexión

Torna a arrencar i comprova que el sistema arrenca amb el nou disc de forma normal.
