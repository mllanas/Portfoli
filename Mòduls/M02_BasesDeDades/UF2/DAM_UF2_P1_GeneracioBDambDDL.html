Pràctica UF2P1 - Generació d'una BD amb DDL

Realitzar les tasques per adquirir els coneixements per aprendre a implementar una base de dades amb llenguatge SQL, on a partir d’un problema realitzarem la transformació del disseny ER a Model Relacional i posterior BD SQL, a partir dels recursos treballats durant l’activitat d’ensenyament-aprenentatge A01, A02, A03 i A04. A cada una d’elles, s’ha de realitzar les tasques oportunes per verificar el seu procediment de resolució.

Documentació a lliurar

  • un document en PDF anomenat "UF2P1.pdf" que contingui el diagrama E-R (Tasca 1) i el model relacional (Tasca 2).
  • un ﻿arxiu de text anomenat "UF2P1.sql" que contingui les sentències DDL demanades a la Tasca 3.

L'arxiu ha de ser anònim i no ha de contenir ni el vostre nom ni cap altre dada que us identifiqui.

Enunciat

Volem emmagatzemar la informació dels usuaris que es connecten als diferents ordinadors que tenim en les aules d’una universitat. Un ordinador s’identifica per l’aula on es troba (A01, A02, B03...) i per un número correlatiu dins l’aula (1, 2, 3...).

Dels ordinadors ens interessa saber-ne el número de sèrie (que és únic), la velocitat (en format numèric amb dos decimals) i la memòria (exemple: 256 MB, 1 GB, etc.). Identifiquem els usuaris per un nom d'usuari, però també emmagatzemem el seu DNI (que és únic) i el seu nom, cognoms, correu electrònic (hem de mirar que el valor contingui el caràcter @) i població. Quan un usuari es connecta a un ordinador emmagatzemem l'usuari que ha fet la connexió, la data i hora de connexió, per defecte la data d’avui.

Temps estimat: 2h.

Avaluació: solució a moodle.

Tasca 1. Diagrama Entitat-Relació

Realitza el diagrama ER equivalent. A més, anota amb la representació amb text cadascuna de les entitats.

Tasca 2. Model Relacional

Realitza la traducció del diagrama Entitat-Relació al Model Relacional.

AULA (codi_aula)


USUARI (codi_usuari, login, dni, nom, cognoms, email, poblacio)

ORDINADOR (codi_ordinador, numero_serie, velocitat, memoria, codi_aula)

        on {codi_aula} referencia a AULA

UTILITZA (codi_usuari, codi_ordinador, codi_aula, data_connexio)

        on {codi_usuari} referencia a USUARI

        on {codi_ordinador, codi_aula} referencia a ORDINADOR

Tasca 3. DDL

Genera el script SQL amb les sentències DDL de creació del model. S’ha de tenir en compte que s’ha de crear una BD nova, i un cop creades les taules i les relacions, es vol crear:

  1. Un índex pel correu electrònic de l’usuari (CREATE).
  2. Un índex per la població de l’usuari (ADD).

Nota: Crea les restriccions (constraints) que siguin necessàries, segons les guies de bones pràctiques proposades a Moodle (Coding Standards i Constraint Coding).

DROP DATABASE universitat;

CREATE DATABASE universitat;

USE universitat;

 

DROP TABLE IF EXISTS usuari_ordinador;

DROP TABLE IF EXISTS aula;

DROP TABLE IF EXISTS usuari;

DROP TABLE IF EXISTS ordinador;

USE universitat;

CREATE TABLE aula (

id_aula                CHAR(3),

PRIMARY KEY (id_aula)

);

 

CREATE TABLE usuari (

id_usuari        SMALLINT AUTO_INCREMENT,

login                VARCHAR(20),

dni                        CHAR(9),

nom                        VARCHAR(20),

cognoms                VARCHAR(40),

email                VARCHAR(50),

poblacio        VARCHAR(50),

PRIMARY KEY (id_usuari),

CONSTRAINT uk_usuari_dni UNIQUE (dni),

CONSTRAINT ck_usuari_email CHECK (email LIKE '%_@__%.__%')

);

 

CREATE TABLE ordinador (

id_ordinador        TINYINT(2) AUTO_INCREMENT,

numero_serie        VARCHAR(15),

velocitat                DECIMAL(3,2),

memoria                        VARCHAR(10),

aula_id                        CHAR(3) NOT NULL,

PRIMARY KEY (id_ordinador, aula_id),

CONSTRAINT uk_ordinador_numero_serie UNIQUE (numero_serie),

CONSTRAINT fk_ordinador_aula FOREIGN KEY (aula_id)

        REFERENCES aula(id_aula)

);

 

CREATE TABLE usuari_ordinador

(

usuari_id                SMALLINT,

ordinador_id        TINYINT(2),

aula_id                        CHAR(3),

data_connexio        DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,

PRIMARY KEY (usuari_id, ordinador_id, aula_id),

CONSTRAINT fk_usuari_ordinador_usuari FOREIGN KEY (usuari_id)

        REFERENCES usuari(id_usuari),

CONSTRAINT fk_usuari_ordinador_ordinador FOREIGN KEY (ordinador_id, aula_id)

        REFERENCES ordinador(id_ordinador, aula_id)

);

CREATE INDEX ix_usuari_email ON usuari(email);

ALTER TABLE usuari ADD INDEX ix_usuari_poblacio(poblacio);

SHOW INDEX FROM usuari;
